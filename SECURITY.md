# Security Policy

Thank you for helping keep HtmlForge secure.

## Reporting a Vulnerability
- Email: k.yagci@students.htl-leonding.ac.at
- Include: clear description, affected area(s)/version(s), steps to reproduce, impact, and suggested remediation if known.
- Optional: CVSS score and a minimal proof-of-concept. Please do not publicize details until a fix is released.

## Response Timelines
- Acknowledgement: within 3 business days.
- Initial assessment: within 7 days where feasible.
- Fix/Mitigation: prioritized by severity; we will keep you informed and credit you if you wish.

## Scope
- Vulnerabilities in the HtmlForge codebase, releases, and distributed packages.

## Supported Versions
- We support security fixes for the main branch and the latest released version.
- Older versions may receive fixes on a best-effort basis.

## Preferred Communication
- If you prefer encrypted communication, mention this in your email and we will arrange a secure channel.

## Responsible Disclosure
- Please avoid public disclosure until a fix or mitigation is available. Coordinated disclosure is appreciated.
